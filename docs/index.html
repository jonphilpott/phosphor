Scene Anatomy

A scene is a plain Lua file. The engine calls three optional global functions:

HookWhen calledTypical use
on_load()Once, after the file is executedAllocate canvases, load images, set initial shader
on_frame(dt)Every frame — dt is seconds since last frameClear, draw geometry, advance animation state
on_osc(addr, ...)Once per incoming OSC messageReact to live control from SuperCollider / PD

All three are optional. A scene that defines none of them is valid (the fallback triangle is shown).

Built-in globals

NameTypeDescription
screen_widthintegerDrawable width in pixels (Retina-aware). Updated on resize and fullscreen toggle.
screen_heightintegerDrawable height in pixels.

Minimal scene template

-- Copy-paste starting point

local t = 0

function on_load()
    shader_set("scanlines")
end

function on_frame(dt)
    t = t + dt
    clear(0, 0, 0, 1)
    set_color(0, 1, 0.4, 1)
    draw_circle(screen_width / 2, screen_height / 2, 60 + math.sin(t * 2) * 20)
end

Hot reload

Save the scene file while Phosphor is running — changes appear within ~200 ms. The Lua VM is torn down and rebuilt; GPU state (renderer, FBOs, shader pipeline) is untouched. on_load() is called again after reload.

Drawing API

All coordinates are in pixel space: (0, 0) is the top-left corner, +X rightward, +Y downward. All geometry goes through the current transform matrix (see Transform Stack).

Clear

SignatureDescription
clear(r, g, b, a)Fill the frame buffer with a solid colour. Call this at the start of on_frame unless you want feedback trails.

Colour state

Two independent colour registers. State persists across frames until changed.

SignatureApplies toDefault
set_color(r, g, b, a)draw_rect, draw_circle1, 1, 1, 1
set_stroke(r, g, b, a)draw_line, draw_point1, 1, 1, 1
set_stroke_weight(w)Line thickness / point size in pixels1.0
set_circle_segments(n)Tessellation for draw_circle32

Primitives

SignatureDescription
draw_rect(x, y, w, h)Filled rectangle. (x, y) is the top-left corner.
draw_circle(cx, cy, r)Filled circle centred at (cx, cy) with radius r pixels.
draw_line(x1, y1, x2, y2)Expanded quad line (not glLineWidth). Uses stroke colour + weight.
draw_point(x, y)Square dot centred at (x, y). Size = stroke weight.
function on_frame(dt)
    clear(0.05, 0.05, 0.05, 1)

    set_color(0.2, 0.8, 0.4, 1)
    draw_rect(100, 100, 200, 80)

    set_color(1, 0.3, 0.1, 1)
    set_circle_segments(64)
    draw_circle(400, 300, 50)

    set_stroke(1, 1, 1, 1)
    set_stroke_weight(2)
    draw_line(0, 0, screen_width, screen_height)

    set_stroke_weight(6)
    draw_point(screen_width / 2, screen_height / 2)
end
Transform Stack

The transform stack works like Processing's pushMatrix/popMatrix. Transforms are post-multiplied — they apply in the order you call them.

FunctionDescription
push()Save the current matrix. Pair every push() with a pop().
pop()Restore the matrix saved by the matching push().
translate(x, y)Move the origin by (x, y) pixels.
rotate(radians)Rotate counter-clockwise around the current origin.
scale(sx, sy)Scale around the current origin. Pass one value to scale uniformly.
-- Rotate a rectangle around its own centre
push()
    translate(cx, cy)   -- move origin to rect centre
    rotate(angle)       -- rotate around that point
    draw_rect(-hw, -hh, hw * 2, hh * 2)  -- draw centred at origin
pop()                   -- restore transform
Transform stack does not affect canvas or image draws canvas:draw(), img:draw(), and sheet:draw() use a separate GPU shader that bypasses the CPU transform matrix entirely. push/translate/rotate/scale have no effect on these calls. Use the angle parameter on those functions instead — see Canvas and Images & Sprites.
Feedback

Feedback blends the previous frame's final composited image over the current frame buffer. Call it before drawing geometry in on_frame — it acts as a persistent background that geometry is drawn on top of.

draw_feedback(alpha [, scale [, angle]])
ParameterDefaultEffect
alpha—Blend weight 0–1. Lower = faster fade. Required.
scale1.0Scale the previous frame around the screen centre. >1 = infinite zoom outward; <1 = shrinking spiral.
angle0.0Rotate the previous frame around the screen centre (radians). Non-zero = rotating trail.

Alpha decay reference (at 60 fps)

alphaHalf-lifeCharacter
0.90~0.4 sFast phosphor decay
0.95~1.0 sMedium trail
0.97~1.6 sLong smear
0.99~5.0 sGhost persistence
function on_frame(dt)
    -- No clear() — feedback IS the background
    draw_feedback(0.93, 1.002, 0.002)   -- slow zoom + slow rotate = spiral trail

    set_color(1, 1, 1, 1)
    draw_circle(math.random() * screen_width,
                math.random() * screen_height, 8)
end
Tip Omitting clear() entirely and relying on feedback gives the classic CRT phosphor look. Adding a very dark clear(0,0,0,0.03) as an FBO blend is not directly supported — use alpha < 1.0 on draw_feedback instead.
Post-Process Shader Pipeline

Post-process shaders run on the full frame after geometry is drawn. Shaders are chained: the output of each feeds the next. The pipeline is applied once per frame, in end_frame().

FunctionDescription
shader_set(name, ...)Replace the entire pipeline. Pass multiple names to chain them left-to-right.
shader_add(name)Append one shader to the current pipeline.
shader_clear()Remove all post-process shaders.
shader_set_uniform(name, value)Set a named float uniform on all active shaders.
Common mistake Calling shader_set twice sets only the last pipeline. To chain two shaders, pass both names to a single call:
shader_set("scanlines", "chromatic_ab")

Built-in shaders

NameEffectUniforms
scanlinesDims alternating pixel rows — CRT scanline look.none
chromatic_abSamples R, G, B channels at slightly offset UVs — colour fringing on bright edges.u_chrom_amount (default 0.002)
function on_load()
    shader_set("scanlines", "chromatic_ab")
    shader_set_uniform("u_chrom_amount", 0.004)
end

-- Change aberration amount live
function on_osc(addr, ...)
    local args = {...}
    if addr == "/chrom" then
        shader_set_uniform("u_chrom_amount", args[1])
    end
end

Writing a Custom Shader

Place fragment shader files in shaders/<name>.frag relative to the working directory. Load with shader_set("name") — no extension, no path.

Required preamble

#version 330 core

uniform sampler2D u_texture;
uniform vec2      u_resolution;
uniform float     u_time;
uniform float     u_beat;

in  vec2 v_uv;
out vec4 frag_color;

Minimal passthrough

#version 330 core
uniform sampler2D u_texture;
in  vec2 v_uv;
out vec4 frag_color;

void main() {
    frag_color = texture(u_texture, v_uv);
}

Example: greyscale with vignette

#version 330 core
uniform sampler2D u_texture;
uniform vec2      u_resolution;
in  vec2 v_uv;
out vec4 frag_color;

void main() {
    vec4 c = texture(u_texture, v_uv);
    float grey = dot(c.rgb, vec3(0.299, 0.587, 0.114));

    // Vignette: darken corners
    vec2 uv = v_uv * 2.0 - 1.0;
    float vig = 1.0 - dot(uv, uv) * 0.4;

    frag_color = vec4(vec3(grey * vig), c.a);
}

Custom float uniforms from Lua

-- In Lua:
shader_set("my_shader")
shader_set_uniform("u_my_value", 0.5)

-- In GLSL:
uniform float u_my_value;
Note Shaders are reloaded from disk each time shader_set() or shader_add() is called. You can edit a .frag file and call shader_set from the scene to hot-reload it.
Noise Functions

Classic Perlin noise (Ken Perlin's 2002 improved permutation table). All functions return values in [−1, 1].

SignatureDescription
noise(x)1-D Perlin noise
noise(x, y)2-D Perlin noise
noise(x, y, z)3-D Perlin noise
fbm(x, y [, octaves [, lacunarity [, gain]]])Fractal Brownian Motion — stacked octaves of noise

fbm defaults

ParameterDefaultEffect
octaves6Number of noise layers (1–16)
lacunarity2.0Frequency multiplier per octave
gain0.5Amplitude multiplier per octave
Performance note Noise is a Lua→C call with moderate overhead. Calling it for every pixel of a full-screen grid (e.g. step = 4) will drop below 60 fps on large Retina displays. Use a larger step (12–24 px) for realtime visualisations, or write a custom GLSL shader for full-resolution noise fields — the GPU runs noise at native speed. 
-- Scrolling 2-D noise field
local scale = 0.006
local t = 0

function on_frame(dt)
    t = t + dt * 0.4
    clear(0, 0, 0, 1)
    local step = 18
    for y = 0, screen_height - 1, step do
        for x = 0, screen_width - 1, step do
            local n = noise(x * scale + t, y * scale) * 0.5 + 0.5
            set_color(n, n * 0.8, n * 0.3, 1)
            draw_rect(x, y, step, step)
        end
    end
end
Images & Sprite Sheets

Images are loaded from PNG or JPG files. Load once in on_load() — not inside on_frame().

Transform stack bypass img:draw() and sheet:draw() use draw_image() internally, which runs through a dedicated GPU shader. push/translate/rotate/scale have no effect on these calls. Use the optional angle parameter for rotation.

image

SignatureDescription
image.load("path")Load a PNG or JPG from disk. Returns an Image userdata.
img:draw(x, y [, w, h [, angle]])Draw at pixel position. w/h default to image size. angle in radians, rotates around the draw rect's centre.
img:draw_region(x, y, w, h, sx, sy, sw, sh [, angle])Draw a sub-rectangle of the image. All s* coords are in image pixel space.
img:width(), img:height()Image dimensions in pixels.

sprite_sheet

SignatureDescription
sprite_sheet.new("path", fw, fh)Load a sprite sheet with frame size fw × fh pixels.
sheet:draw(idx, x, y [, w, h [, angle]])Draw one frame. idx is 1-based, left→right then top→bottom. angle rotates around draw rect centre.
sheet:frame_count()Total frames in the sheet.
sheet:cols(), sheet:rows()Grid dimensions.
local img
local sheet
local frame = 1
local t = 0

function on_load()
    img   = image.load("assets/logo.png")
    sheet = sprite_sheet.new("assets/walk.png", 64, 64)
end

function on_frame(dt)
    t = t + dt
    clear(0, 0, 0, 1)

    -- Draw image, spinning slowly
    img:draw(100, 100, img:width(), img:height(), t * 0.5)

    -- Animate sprite sheet at 12 fps
    frame = math.floor(t * 12) % sheet:frame_count() + 1
    sheet:draw(frame, 400, 200, 128, 128)
end
Canvas

A canvas is an offscreen render target (FBO pair) with its own optional local shader chain. Render geometry into it, apply shaders scoped only to that canvas, then composite the result into the main scene.

Allocate canvases in on_load() or at the top level of the scene — not inside on_frame(). Creating an FBO pair every frame leaks GPU memory.

API

SignatureDescription
canvas.new(w, h)Allocate an offscreen canvas at w × h pixels. Returns a Canvas userdata.
canvas:begin()Redirect all subsequent draw calls into this canvas. Pushes the render-target stack.
canvas:finish([shader, ...])Flush geometry, optionally run a local shader chain, pop the render-target stack. Zero or more shader names can be passed.
canvas:draw(x, y [, w, h [, angle]])Blit the canvas result into the current scene. w/h default to canvas size. angle rotates around the draw rect's centre (radians).
canvas:set_uniform(name, value)Set a named float uniform on this canvas's local shader pipeline. Must be called before canvas:finish(). See Fractal Shaders for the primary use case.
canvas:width(), canvas:height()Canvas dimensions in pixels.
Transform stack bypass canvas:draw() uses draw_image() internally and runs through a dedicated GPU shader. push/translate/rotate/scale have no effect on canvas draws. The angle parameter is the only way to rotate a canvas when compositing it. Position and scale are controlled by the x, y, w, h arguments.

Frame flow

function on_frame(dt)
    -- Draw into canvas
    c:begin()
        clear(0, 0, 0, 1)
        set_color(1, 0.4, 0.1, 1)
        draw_circle(c:width()/2, c:height()/2, 80)
    c:finish("chromatic_ab")    -- chromatic_ab applied only to this canvas

    -- Composite into main scene
    clear(0, 0, 0, 1)
    c:draw(100, 50, 400, 300, math.sin(t) * 0.3)   -- positioned, scaled, rotated
end

Canvases nest

A canvas:begin() inside another canvas's begin()/finish() block works correctly — the render-target stack handles save/restore automatically.

Local shader chain

Shaders passed to canvas:finish() are scoped to that canvas only — they do not affect the rest of the scene. The main post-process pipeline (set with shader_set()) runs afterwards on the full composited frame.

Waveforms

Two layers of waveform support: value functions that return a float for use as a modulator, and a renderer that draws a waveform as a polyline into the scene.

In all functions, t is a phase in cycles — 0 = start of period, 1 = one full period, 2 = two periods. Passing t * freq gives you freq oscillations per second.

Value functions — return float in [−1, 1]

SignatureShape
wave_sine(t)Smooth sine. Good for organic modulation.
wave_saw(t)Ramps −1→+1 then jumps. Bright, harsh timbre.
wave_square(t [, duty])+1 for the first duty fraction of the period, then −1. duty defaults to 0.5. Clamped to 0.01–0.99.
wave_tri(t)Rises −1→+1 in first half-period, falls back in second. Softer than saw.

Renderer

draw_waveform(type, x, y, w, h [, cycles [, phase]])
ParameterDefaultDescription
type—One of "sine", "saw", "square", "tri". Error if unknown.
x, y—Top-left corner of the bounding rectangle in pixel space.
w, h—Bounding rectangle dimensions. The waveform is centred vertically; amplitude = h/2.
cycles1.0Number of complete periods drawn across the width.
phase0.0Starting phase in cycles. Pass t here to scroll the waveform over time.

draw_waveform respects the CPU transform matrix — push/translate/rotate/scale apply. Sampling density is one sample per 2 pixels of width, clamped to 4–2048.

local t = 0

function on_frame(dt)
    t = t + dt
    clear(0, 0, 0, 1)

    -- Value function: modulate circle radius with a triangle wave at 0.5 Hz
    local r = (wave_tri(t * 0.5) * 0.5 + 0.5) * 80 + 20
    set_color(0.3, 0.8, 0.4, 1)
    draw_circle(screen_width / 2, screen_height / 3, r)

    -- Renderer: scrolling oscilloscope strip
    set_stroke(0.2, 0.9, 0.4, 1)
    set_stroke_weight(1.5)
    draw_waveform("sine", 0, screen_height * 0.6, screen_width, 120, 3, t)

    set_stroke(0.9, 0.4, 0.1, 1)
    draw_waveform("saw", 0, screen_height * 0.75, screen_width, 120, 3, t * 1.5)
end
3D Wireframe

CPU-side 3D projection using a right-handed Y-up coordinate system. All geometry is drawn as line segments using the current stroke colour and weight. The CPU transform matrix applies — wrap draws in push()/pop() to apply 2D offsets after projection.

Setup

SignatureDescription
camera_3d(ex, ey, ez, tx, ty, tz)Set eye position and look target. Rebuilds the view matrix.
perspective_3d(fov [, near [, far]])Set projection. fov in radians. Aspect ratio derived from screen_width/screen_height automatically. Defaults: fov=1.047 (60°), near=0.1, far=1000.
reset_3d()Restore defaults: eye=(0,3,6) looking at origin, 60° FOV.

Projection

SignatureDescription
sx, sy = project_3d(wx, wy, wz)Project a world-space point to screen pixel coordinates. Returns two values on success, or nothing (zero return values) if the point is behind the camera. Always check: if sx then ... end.

Shapes

SignatureDescription
draw_wire_cube(cx, cy, cz, size, rx, ry, rz)Wireframe cube centred at (cx,cy,cz). rx/ry/rz are Euler rotation angles in radians (applied Ry·Rx·Rz).
draw_wire_sphere(cx, cy, cz, r [, lat [, lon]])Latitude rings + longitude arcs. Defaults: lat=8, lon=16.
draw_wire_grid(size, divs [, y])XZ-plane grid centred at the origin at height y (default 0). size is total width/depth in world units; divs divisions per axis.
Performance note Projection is done on the CPU in Lua→C calls. For dense point clouds (thousands of points per frame), consider batching in a Lua table and projecting only visible points. Spheres with many segments (lat > 16, lon > 32) can get expensive — 8/16 looks good at most sizes. 
local t = 0

function on_load()
    perspective_3d(1.047, 0.1, 200.0)
end

function on_frame(dt)
    t = t + dt
    clear(0.03, 0.03, 0.03, 1)

    -- Orbit camera around the scene
    camera_3d(math.cos(t * 0.3) * 8, 3, math.sin(t * 0.3) * 8,   0, 0, 0)

    set_stroke(0.15, 0.3, 0.15, 1)
    draw_wire_grid(8, 8, 0)

    set_stroke(0.3, 0.9, 0.4, 1)
    set_stroke_weight(1.5)
    draw_wire_cube(0, 1, 0,  1.2,  t * 0.7, t * 0.4, 0)

    set_stroke(0.3, 0.6, 1.0, 1)
    draw_wire_sphere(2, 1, 0,  0.8,  8, 16)

    -- Manual projection: draw a point cloud
    set_stroke(1.0, 0.7, 0.1, 1)
    set_stroke_weight(4)
    for i = 0, 23 do
        local a = (i / 24) * math.pi * 2 + t
        local sx, sy = project_3d(math.cos(a) * 2, 1.5, math.sin(a) * 2)
        if sx then draw_point(sx, sy) end
    end
end
Fractal Shaders

Two built-in fractal fragment shaders. Both ignore u_texture entirely — they generate the full image from the uniforms and u_time. They are most useful as:

  • Full-screen post-process: shader_set("mandelbrot") — replaces the frame entirely.
  • Canvas-local: render into a canvas via canvas:finish("julia") with uniforms set via canvas:set_uniform(), then composite the canvas into a larger scene.
Important If you use a fractal shader in canvas:finish() without calling canvas:set_uniform("u_zoom", ...), u_zoom defaults to 0 in GLSL. The shader clamps it to 0.0001, which maps the entire Mandelbrot/Julia set onto a microscopic region — the result is a near-uniform colour. Always set u_zoom to at least 1.0.

mandelbrot

Classic Mandelbrot set. Pan with u_center_x/y, zoom with u_zoom.

UniformDefaultDescription
u_center_x0.0Real axis pan. Try −0.75 for the classic seahorse valley view.
u_center_y0.0Imaginary axis pan.
u_zoom0→1.0Zoom level. 1.0 = full set visible. Higher = deeper zoom.
u_max_iter0→32Iteration cap. Higher = sharper boundary, slower GPU. 64–256 is typical.
u_color_shift0.0Phase offset into the cosine palette. Animating this cycles colours.
u_color_speed0.05Automatic palette cycle speed driven by u_time. Set to 0 if driving u_color_shift manually.

julia

Julia set. The complex constant c = (u_julia_cx, u_julia_cy) determines the shape. When both are 0 (GLSL default), the shader falls back to c = (−0.7, 0.27) — the Siegel disk.

UniformDefaultDescription
u_julia_cx−0.7*Real part of c. *Fallback only when both cx and cy are 0.
u_julia_cy0.27*Imaginary part of c.
u_zoom0→1.0As above.
u_max_iter0→32As above.
u_color_shift0.0As above.
u_color_speed0.05As above.
u_animate0.0If > 0, orbits c around (u_julia_cx, u_julia_cy) at this radius, driven by u_time. Set to 0.3–0.5 for continuous morphing.

Interesting Julia c values

cxcyShape
−0.700.27Siegel disk (default) — symmetric, lacy
−0.750.11Douady rabbit — three-lobed
−0.120.74Dendrite — near Misiurewicz point, spiky
0.280.01Fat fractal — filled, blobby

Full-screen example

local t = 0

function on_frame(dt)
    t = t + dt
    clear(0, 0, 0, 1)

    shader_set("julia")
    shader_set_uniform("u_julia_cx",    -0.75)
    shader_set_uniform("u_julia_cy",     0.11)
    shader_set_uniform("u_zoom",         1.3)
    shader_set_uniform("u_max_iter",     128.0)
    shader_set_uniform("u_color_shift",  t * 0.03)
    shader_set_uniform("u_color_speed",  0.0)
    shader_set_uniform("u_animate",      0.0)
end

Canvas example (fractal as background layer)

local c_frac

function on_load()
    c_frac = canvas.new(math.floor(screen_width), math.floor(screen_height))
end

function on_frame(dt)
    t = t + dt
    clear(0, 0, 0, 1)

    -- Render Julia into canvas with full uniform control
    c_frac:begin()
        clear(0, 0, 0, 1)
    c_frac:set_uniform("u_zoom",        1.3)
    c_frac:set_uniform("u_max_iter",    96.0)
    c_frac:set_uniform("u_color_shift", t * 0.02)
    c_frac:set_uniform("u_color_speed", 0.0)
    c_frac:set_uniform("u_animate",     0.3)
    c_frac:finish("julia")

    -- Draw as background, then composite geometry on top
    c_frac:draw(0, 0, screen_width, screen_height)

    -- Geometry drawn here sits on top of the fractal
    set_stroke(1, 1, 1, 0.8)
    set_stroke_weight(1.5)
    draw_wire_cube(0, 1, 0, 1.2, t, t * 0.5, 0)
end
OSC

Phosphor listens on UDP port 9000. Multiple clients (SuperCollider, Pure Data, TouchOSC, Max) send to the same port simultaneously — UDP is connectionless.

Incoming messages are dispatched on the main thread once per frame. The on_osc hook is called once per message.

function on_osc(addr, ...)
ArgumentTypeDescription
addrstringOSC address path, e.g. "/color"
...numbers / stringsOSC int and float arguments arrive as Lua numbers; OSC strings arrive as Lua strings. Zero or more.

Example

function on_osc(addr, ...)
    local args = {...}
    if addr == "/color" then
        r, g, b = args[1], args[2], args[3]
    elseif addr == "/speed" then
        speed = args[1] or speed
    elseif addr == "/bang" then
        flash = true
    end
    -- Unknown addresses are silently ignored — no crash, no error.
end

Sending from SuperCollider

~p = NetAddr("127.0.0.1", 9000);

// Single message
~p.sendMsg("/color", 1.0, 0.0, 0.5);

// Routine for continuous control
(
r = Routine({
    loop {
        ~p.sendMsg("/speed", rrand(0.5, 2.0));
        0.5.wait;
    };
}).play(AppClock);
)
r.stop;

Engine-level address

One OSC address is handled by the engine itself and is never forwarded to on_osc:

AddressArgumentEffect
/scenestring — file pathLoad a new scene immediately. Path is relative to the working directory. Works from any client, even if the current scene has no on_osc.
-- SuperCollider
~p.sendMsg("/scene", "scenes/matrix.lua");
~p.sendMsg("/scene", "scenes/fractal_test.lua");
Note If on_osc is not defined in the scene, all non-engine messages are silently discarded.
